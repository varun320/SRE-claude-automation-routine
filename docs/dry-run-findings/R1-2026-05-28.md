## What needs fixing before R1 ships to cloud cron 🔧

### Fix-1 (HIGH) — D5 auto-archive ruleset too narrow

The current ruleset requires all three of: (a) sender domain ∈ allowlist, (b) subject matches marketing regex, (c) bulk send. In the window:

- Real bulk-marketing messages (6 of them, ~27% of window volume) **didn't match the regex** — their subjects use vendor-specific phrasing the regex doesn't catch.
- All `linkedin.com` noise (5 messages, ~23% of window) was already dropped by D6 before D5 saw it — D5 effectively only fires on noise addressed directly to Maaz, which is the rare case.

**Recommended fix (Recommendation B in issues file):**

Replace the regex-based D5 with an **LLM-classified 4th bucket: ARCHIVE**. The classifier already reads sender + subject for RED/AMBER/GREEN bucketing — let it also flag transactional / bulk-marketing / promotional intent. Operator reviews the ARCHIVE bucket via dry-run logs; once trust is established (e.g., 50 ARCHIVE classifications with 0 false-positives), the routine actually moves them.

This avoids a regex-maintenance treadmill and matches how Maaz would triage himself.

Affects: `routines/tier1/R1-na-inbox-scan.md` D5 section, `routines/tier1/R5-me-inbox-scan.md` D5 section (mirror change).

### Fix-2 (MEDIUM) — Empty `toRecipients` BCC edge case

2 messages had `toRecipients: []` — both legitimate Maaz-recipient mail (ME-region customer + tender platform), BCC pattern. The D6 filter dropped them as "not addressed to maaz/info/info-me."

**Recommended fix:** add a fallback in D6 — if `toRecipients` is empty AND the message is in Maaz's primary mailbox, keep it with annotation `[delivered via BCC]`. Cross-mailbox-pollution risk is low: Don's BCC items live in Don's mailbox, not Maaz's.

Affects: R1 + R5 D6 filter logic.

### Fix-3 (LOW) — Smoke-test required before enabling actual archive moves

`list-mail-folder-messages` returned messages addressed to Don / Husam / external BCC patterns — origin (Maaz's own inbox vs aggregated shared-mailbox view) is unclear. Before D5 ever actually calls `move-mail-message`, we need to confirm whether it moves Maaz's copy or the original in Don's mailbox.

**Recommended fix:** add a smoke-test step to `connectors/connector-check.md` — move one self-sent message to `Archive/Test/`, verify only Maaz's copy moved, revert.

Affects: `connectors/connector-check.md`, gates OPEN-H.
