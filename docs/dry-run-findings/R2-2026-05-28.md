## BLOCKING — Mac-path inputs are unreachable from cloud

R2's build prompt reads from `/Users/maazwork/Documents/Claude/Projects/SRE General Manager/`:
- `Action Package {Month YYYY}.md` (top input)
- `Task List`
- `Daily Logs/{yesterday}.md`

These live on Maaz's Mac. They are **not reachable from**:
- This Windows dry-run host
- Anthropic's cloud runtime (the only filesystem the cloud agent gets is its ephemeral worktree)

**Implication:** R2 in its current form cannot run as a cloud cron. Without those inputs, Top 3 generation degrades to "pick three things from today's calendar" — which is useless when today's calendar is empty (today's case).

**Three resolution paths, ranked:**

| Option | Effort | Quality of R2 output |
|---|---|---|
| **A. Migrate Action Package + Task List + Daily Logs to OneDrive** — Maaz updates the live SRE GM project to reference OneDrive paths; cloud agent reads via Graph `Files.Read.All` (already in routine scopes). | Medium — requires Maaz to change his working flow OR set up a sync between Mac path and OneDrive. | Best — R2 has full input fidelity. |
| **B. Migrate only a slim daily-input file to OneDrive** — Maaz writes a short `routine-inputs/{YYYY-MM-DD}.md` to OneDrive each morning summarizing today's themes + must-hit items. R2 reads only that. | Low — single file per day, can be hand-written or scripted. | Good — explicit operator framing, less LLM inference. |
| **C. Drop Mac-path read from R2 entirely** — R2 becomes "calendar + R4 daily-close (when R4 ships) + light memory only." Acknowledge weaker Top 3 output. | None | Weaker — Top 3 becomes generic on days with sparse calendars (most days, given today's empty result). |

**Recommendation: Option B for v0.** Lowest friction; preserves R2's Top 3 quality; doesn't require restructuring Maaz's existing SRE GM workflow.

This is **Fix-8 (HIGH)** in the master fix list.

## Observation 1 — Bi-Weekly Job not visible on this Thursday

D9 says the canonical observed recurring meeting is **Thursday 10:00 MT Bi-Weekly Job**. Today is Thursday. The calendar shows zero events today.

**Possible explanations:**
- Today is the "off" Thursday in the bi-weekly cycle (every other Thursday → could have been last Thursday, will be next Thursday). Plausible.
- The Bi-Weekly Job lives on a different calendar (e.g., a team calendar Maaz attends but doesn't organize). Less likely — list-calendars shows only 5 attached.
- The meeting was moved or cancelled this week. Possible.

**Action:** before R2 cloud cron, verify the Bi-Weekly Job's actual recurrence pattern from the past 3-4 occurrences. If "bi-weekly" means every-other-Thursday, R2 needs to know which Thursday is "on" vs "off" to surface the meeting in advance.

**Status:** OPEN-M (operational observation, not a spec bug).

## Observation 2 — Zoho CRM Calendar still attached
